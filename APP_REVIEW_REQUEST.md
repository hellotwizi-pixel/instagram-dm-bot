# Meta 앱 검수 요청서 (제출용 초안)

Meta 개발자 콘솔의 "앱 검수" 화면에서 각 권한(permission)마다 "왜 이 권한이 필요한지" 설명을 영어 또는 한국어로 적어야 합니다. 아래 내용을 그대로 복사해서 채우면 됩니다. 제출 전에 실제 콘솔에 떠 있는 권한 이름과 아래 목록이 같은지만 한 번 확인하세요.

---

## 1. 한 줄 사용 목적 (앱 설명)

```
이 앱은 인스타그램 비즈니스 계정의 게시물 댓글에서 특정 키워드를 감지하여,
댓글 작성자에게 비공개 다이렉트 메시지(DM)로 미리 준비된 안내 자료(가격표, 구독 안내 등) 링크를
자동으로 전송하는 고객 응대 자동화 도구입니다. 일반 사용자에게 노출되는 화면은 없으며,
비즈니스 계정 운영자가 서버를 통해 내부적으로만 사용합니다.
```

## 2. 권한별 사용 목적 설명

### instagram_business_basic
```
연동된 인스타그램 비즈니스 계정의 기본 정보(계정 ID 등)를 확인하여,
어느 계정에 달린 댓글인지 식별하는 데 사용합니다.
```

### instagram_business_manage_comments
```
게시물에 달린 댓글의 내용을 webhook으로 수신하여, 미리 등록된 키워드와 일치하는지
확인하는 데 사용합니다. 댓글 내용은 키워드 매칭 목적으로만 사용되고 별도 저장되지 않습니다.
```

### instagram_business_manage_messages
```
키워드가 일치한 댓글 작성자에게 비공개 답장(private reply) 형태로 안내 자료 링크가 담긴
다이렉트 메시지를 전송하는 데 사용합니다. 발신 메시지는 사전에 운영자가 등록한 고정 문구만
사용하며, 임의의 메시지를 작성하거나 대량 발송하지 않습니다.
```

> 콘솔에 다른 권한이 추가로 떠 있다면(예: pages_show_list, pages_manage_metadata 등),
> "페이지와 연동된 인스타그램 계정 정보를 확인하기 위해 필요" 정도로 동일한 톤으로 작성하면 됩니다.

## 3. 데모 영상(스크린캐스트) 시나리오

Meta는 권한마다 "실제로 그 기능을 쓰는 화면"을 녹화한 영상(보통 90초~3분)을 요구합니다.

**중요**: 검수 전(Development Mode)에는 일반 댓글에 대한 webhook 실시간 수신과 댓글 내용 조회 API가
구조적으로 제한되어 있어서, "댓글 작성 → 자동 수신 → DM 발송"이 한 화면에서 끊김 없이 이어지는 영상은
찍을 수 없습니다. 대신 같은 데이터로 각 단계가 정상 작동함을 증명하는 방식으로 녹화합니다.

1. (운영 설정) 키워드/답장 설정 화면(구글시트 또는 코드 설정)을 보여준다 — "오일" 키워드와 안내 자료 링크가 등록되어 있음을 보여줌
2. (사용자 행동) 실제 인스타그램 게시물에 "오일"이라고 댓글을 남긴다 — 실제 계정으로 실제 댓글 작성
3. (서버 동작 증명) Meta 개발자 콘솔의 Webhook "테스트" 기능으로 동일한 키워드("오일")가 담긴 샘플 데이터를 서버로 전송하고,
   Vercel 로그 화면에서 다음이 순서대로 찍히는 것을 보여준다:
   - 댓글 수신 (`Processing comment`)
   - 댓글 텍스트 확인 (`Comment text: "오일"`)
   - 키워드 규칙 매칭 (`Matched rule - sending private reply`)
   - DM 발송 API 호출 시도
4. 자막/음성으로 설명: "검수 전이라 실시간 webhook 수신은 제한되어 있지만, 동일한 구조의 데이터로
   서버가 키워드를 감지하고 DM 발송 로직을 정상적으로 실행하는 것을 확인할 수 있습니다.
   검수 완료 후에는 실제 댓글 작성 시 이 과정이 자동으로, 끊김 없이 실행됩니다."

> 참고: 3번 단계에서 사용하는 샘플 댓글 ID는 Meta가 제공하는 테스트용 가짜 ID라서, 로그에 보이는
> "발송 시도"가 실제 발신 단계에서 한 번 더 실패(Object ID does not exist)할 수 있습니다.
> 이건 정상입니다 — "로직이 거기까지 정확히 도달했다"는 것이 핵심이고, 자막으로 이 점을 짚어주면 됩니다.

## 4. 함께 준비해야 할 항목

- [ ] 개인정보처리방침 URL: `https://instagram-dm-bot-t16o.vercel.app/privacy.html` (이미 배포 완료, 앱 설정에 등록 완료)
- [ ] 앱 아이콘, 앱 이름 등 기본 정보가 비어있지 않은지 확인
- [ ] 테스터 계정으로 위 데모 흐름이 실제로 작동하는 영상 1개
- [ ] 각 권한 설명 글 (위 2번 내용)

## 5. 검수 소요 기간

보통 **3~7 영업일** 정도 걸립니다. Meta 쪽에서 영상이나 설명이 불충분하다고 판단해 추가 정보를 요청하면
한 번 더 답변하고 기다려야 해서, 그런 경우 **2주 이상** 걸릴 수도 있습니다. 가장 흔한 반려 이유는
"권한이 실제로 왜 필요한지 설명이 부족함"과 "데모 영상에 해당 기능이 명확히 안 보임"이므로,
위 시나리오대로 키워드 댓글 → DM 수신 장면이 화면에 분명히 보이게 찍는 게 통과율을 높이는 핵심입니다.
